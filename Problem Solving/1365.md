## 📝 Problem Statement

Given an array `nums`, for each `nums[i]` find out how many numbers in the array are **strictly smaller** than it. Return the result as an array.

---

## 📥 Inputs

- `nums` → array of integers

Example:
- `nums = [8, 1, 2, 2, 3]`

---

## 📤 Output

An array where each index `i` contains the count of numbers smaller than `nums[i]`.

Example:
- `[4, 0, 1, 1, 3]`

---

## ⚠️ Constraints

- `2 ≤ nums.length ≤ 500`
- `0 ≤ nums[i] ≤ 100`

---

## 🔍 Examples

### Example 1

Input
```
nums = [8, 1, 2, 2, 3]
```

Output
```
[4, 0, 1, 1, 3]
```

Explanation:
- `8` → 4 numbers smaller (1, 2, 2, 3)
- `1` → 0 numbers smaller
- `2` → 1 number smaller (1)
- `2` → 1 number smaller (1)
- `3` → 3 numbers smaller (1, 2, 2)

### Example 2

Input
```
nums = [6, 5, 4, 8]
```

Output
```
[2, 1, 0, 3]
```

## 💡 Initial Thoughts

- Brute force: nested loops, count directly
- Sorting: after sorting, index = count of smaller elements
- Counting sort: values are bounded 0–100, use frequency + prefix sum

---

## 🧪 Edge Cases

- All elements equal → all zeros
- Single element → `[0]`
- Already sorted ascending → `[0, 1, 2, ...]`
- Already sorted descending → `[n-1, n-2, ..., 0]`
- Contains `0` → some results will be `0`

---

## 🚀 Possible Approaches

### Approach 1 — Brute Force

**Idea:** For every element, loop through the whole array and count elements that are strictly smaller.

**Time Complexity**
```
O(n²)
```

**Space Complexity**
```
O(n) — output array only
```

---

### Approach 2 — Sort + Hash Map

**Idea:** Sort a copy of the array. The index of each element in the sorted array equals the count of smaller numbers. Store the first occurrence of each value in a hash map (to handle duplicates), then map original elements back.

**Time Complexity**
```
O(n log n)
```

**Space Complexity**
```
O(n)
```

---

### Approach 3 — Counting Sort (Optimal) ⚡

**Idea:** Since values are bounded `0–100`, build a frequency array of size 101. Compute a prefix sum so that `prefix[i]` = count of numbers less than `i`. Map each original element using the prefix array.

**Time Complexity**
```
O(n + k) where k = 101 → effectively O(n)
```

**Space Complexity**
```
O(k) → O(1) extra (ignoring output)
```

---

## ✅ Chosen Approach

**Counting Sort** is optimal here because values are bounded to `0–100`, making comparison-based sorting unnecessary.

Step-by-step logic:
1. Build a frequency array `count[0..101]`, incrementing `count[val + 1]` for each value (shift by 1)
2. Compute prefix sums on `count` so `count[i]` = how many numbers are less than `i`
3. For each original element, look up `count[nums[i]]` to get the answer

---

## 🧩 Algorithm (Pseudo Code)

```
1. Create count array of size 102, all zeros
2. For each num in nums: count[num + 1]++
3. For i from 1 to 101: count[i] += count[i - 1]
4. Return nums.map(num => count[num])
```

---

## 💻 Implementation

### Approach 1 — Brute Force

```javascript
var smallerNumbersThanCurrent = function(nums) {
    return nums.map((num, i) =>
        nums.reduce((acc, val, j) => i !== j && val < num ? acc + 1 : acc, 0)
    );
};
```

### Approach 2 — Sort + Hash Map

```javascript
var smallerNumbersThanCurrent = function(nums) {
    const sorted = [...nums].sort((a, b) => a - b);
    const rank = new Map();

    sorted.forEach((val, i) => {
        if (!rank.has(val)) rank.set(val, i); // first occurrence = count of smaller
    });

    return nums.map(num => rank.get(num));
};
```

### Approach 3 — Counting Sort ✅

```javascript
var smallerNumbersThanCurrent = function(nums) {
    const count = new Array(102).fill(0);

    for (const num of nums) {
        count[num + 1]++;            // shift by 1 for easier prefix sum
    }

    for (let i = 1; i < 102; i++) {
        count[i] += count[i - 1];   // prefix sum: count[i] = how many nums < i
    }

    return nums.map(num => count[num]);
};
```

## 📊 Complexity Analysis

| Approach | Time | Space |
|---|---|---|
| Brute Force | O(n²) | O(n) |
| Sort + Hash Map | O(n log n) | O(n) |
| Counting Sort | O(n) | O(1) |

---

## 🧠 Mistakes / Debug Notes

- In Sort + Hash Map: must record **first occurrence only** — duplicates should NOT overwrite the map entry, otherwise equal elements would count each other as smaller
- In Counting Sort: shift index by `+1` when filling (`count[num + 1]++`) so the prefix sum at index `num` naturally represents numbers **strictly less than** `num`

---

## 📚 What I Learned

- Prefix sum on a frequency array is a powerful and reusable pattern
- When value range is small and bounded, counting sort beats O(n log n) sorting
- The `+1` shift trick turns "less than" into a simple prefix lookup
